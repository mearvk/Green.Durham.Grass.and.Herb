MODULE RATIONALE
----------------

This module provides the Appree contact server, labor/ethical/moral/mortality
concerns database system, and coast listeners as a sub-module of the
NitroWebExpress™ / JWSTFJ21 platform. It integrates via reflective masquerade
into the NIO extended port space (0-1048576), registering its listeners under
virtual port assignments that exceed the TCP 16-bit limit.

Appree is a careful and concernful program all the way down to the Mayor of
Chapel Hill and her study of students of the County. This is subtly an Appree
project riding on NC Labor Laws & Organization.

PRIVATE-TO-MASQUERADE TRANSITION (GDGH PERSPECTIVE)
-----------------------------------------------------

Green.Durham.Grass.and.Herb was originally designed as a private, standalone
server suite — local-only listeners bound to physical TCP ports, direct MySQL
access, no awareness of an outer hosting framework.

The transition to masquerade-enabled operation introduces the following local
additions from this program's perspective:

  1. STANDALONE (PRIVATE) MODE — default when JWSTFJ21 is absent.
     - All listeners bind directly to physical ports (2000, 20000, 40002, etc.)
     - BaseListener handles raw TCP connections with its own NationalID gate.
     - No dependency on any external port registry or virtual channel layer.
     - Full functionality retained: geo lookup, DB logging, module install.

  2. MASQUERADE (INTEGRATED) MODE — activated when JWSTFJ21 is on classpath.
     - JWSTFJ21Masquerade reflectively binds to the parent PortRegistry.
     - Physical listeners remain running (backward compatibility).
     - Virtual port registrations are added ON TOP of physical ports:
         Physical 20000 still listens AND virtual 70000 is registered.
     - Inbound traffic may arrive via either path; both route to the same
       handler logic.
     - Module announces itself to the JWSTFJ21 directory server (port 2000
       on the parent), making it discoverable via <nwe-route> XML forwarding.

  3. WHAT CHANGES LOCALLY DURING TRANSITION:
     - source-code/integration/JWSTFJ21Masquerade.java is compiled and loaded.
     - configuration/jwstfj21-integration.xml is parsed for virtual port map.
     - Main.java startup sequence gains a JWSTFJ21 integration phase after
       the standard system-ready check.
     - The program reports INTEGRATED or STANDALONE status at boot.
     - No existing listener code is modified — masquerade wraps, not replaces.

  4. DUAL-HOMED IDENTITY:
     - From JWSTFJ21's perspective: this is module "green-durham-grass-and-herb",
       Rank 4, reachable at virtual ports 70000-99152.
     - From this program's perspective: it is still a private Appree server that
       happens to also be discoverable through a parent NIO masquerade layer.
     - The NationalID gate, Moral Rating checks, and Installer ID requirements
       remain enforced locally regardless of how traffic arrives.

  5. FALLBACK BEHAVIOR:
     - If JWSTFJ21 classes disappear from classpath (jar removed, server down),
       the masquerade deregisters cleanly and the program continues in private
       standalone mode with zero degradation.
     - Retry policy (30s interval, 10 max) allows re-integration if JWSTFJ21
       comes back online without restarting this program.


JWSTFJ21 INTEGRATION PROTOCOL
-------------------------------

1. On startup, Main.java loads configuration/jwstfj21-integration.xml.
2. JWSTFJ21Masquerade is instantiated reflectively.
3. Class.forName() attempts to locate:
     com.mearvk.jwstfj21.PortRegistry
     com.mearvk.jwstfj21.Module
     com.mearvk.jwstfj21.VirtualChannel
4. If found: registers this module's handlers under virtual ports via
   PortRegistry.registerVirtualPort() and announces via announceModule().
5. If not found: degrades to standalone mode, retries per retry policy
   (default: every 30s, max 10 attempts).

XML forwarding from JWSTFJ21 port 2000 directory server:
  <nwe-route><port>70000</port><payload>IDENTIFY</payload></nwe-route>
  Routes to this module's BaseListener (local 20000).
